The Script Editor
 

The scripting inspector or editor enables the user to review the effects of earlier data processing operations, to restore an earlier state and continue processing from that point, or to save a sequence of operations for the purpose of applying it to another trace.

When a file containing unprocessed data is first loaded, the script inspector will display as the first tool "Raw Data". As data processing tools are applied to the data set, the names of the tools will be appended to the list. To view the state of the data after an earlier processing step, click in the far left column next to the tool name. To view the parameters used by an earlier processing step, click on the tool name. The parameters are then displayed in the tool inspector.

Data processing tools always act on the currently displayed data state, so if a processing step in the middle of the script is selected, additional steps will be appended from that point. The current processing step of tool execution in a script is depicted by the blue arrow. A gray arrow indicates the tools are executing and blue indicates execution is complete show the current tool.

The user can click on the Save button at any time to save the currently displayed sequence of processing steps. The filename extension for scripts is ".scr". By selecting Public or Private the scripts will be saved to /Library/BaseFinder/Scripts or to the user library, ~/Library/BaseFinder/Scripts, respectively.

The Load button allows the user to select a saved script to load and apply to the current data set. If any processing has been performed on the current data, that processing will be discarded.

Once a script has been created for one trace, it will typically be suitable for the other traces of the same run. Press the Apply to all lanes button to apply the displayed script to all the lanes in the currently loaded file.
