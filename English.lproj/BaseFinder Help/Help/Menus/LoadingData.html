Loading and Saving Data
 

Data Formats Available

Currently the program can load data in the following formats:

•lanes:  Multi-lane directory format.
•abi:    Processed ABI sequencer format including ab1, abi, abd and fsa formats.
•scf:  SCF format
•dat, txt:  ASCII based data. The data is columned so the number of data points on each line of the file represents the number of channels, and each line is a separate sample point across those channels. Beckman Sequencer raw data exported as a .txt is supported.
•esd:  Beckman format.
•shape:  This format saves the complete script of tools executed on a data set and the associated tool parameters. The original raw data and processed data to the point of the last tool executed are also saved with this format. The calculated alignment and fitted curve information specific to the Align and Integrate tool is also saved.

Loading Data

There are two ways data can be loaded into the program. After launching the program, you can open a data file from the file menu. The program determines the file format to use based on the file extension. The program will also automatically launch and load data when a data file is double clicked in the workspace. In addition, one can drag and drop a data file onto the application.

Currently the program will fail to load a data file if any problems are encountered. The program will probably not crash, but an error message may be presented.

The same data file can be loaded in more than once and hence appear in multiple windows. While this generally does not create problems, if you are saving work from two instances of the same file, only the last save from all of the opened files will actually be saved.

Saving Data

After modifying the trace data it is possible to save the data into the same file or into a new format. The save panel will allow selection from the many options to save the trace data. See above for the different file types.

After the data has been base-called, and the base-sequence window has appeared, it is possible to save out just the base sequence into an ASCII text file.

To do this, simply highlight the base-sequence window and either hit cmd-s or save under the File menu button. A save panel will appear. Upon selecting a directory and entering a file name, the sequence will be saved with the file extension .seq. These file can then be loaded into Edit or any other text viewing program to be printed or viewed.

In summary, BaseFinder can save data in the following formats: .lanes, .data, .seqd, .scf, .txt, and .bfd. The .txt, .dat and .bfd formats each save only the selected lane; the .lanes format saves all lanes. The save format is selected using the pop-up list in the save panel.

There are additional write-only formats: .seq, which is an ASCII format that includes only base sequence data as described above. It can also be selected from the save panel. Also, .fasta which saves the sequence data in FASTA format. It can also be selected from the save panel.

Printing

The data can be printed simply by choosing Print from the menu. Within the printing options panel that pops up, there is an option called BaseFinder. Select this option to set view magnification, the number of views, and whether or not a header should be included in the printout.
