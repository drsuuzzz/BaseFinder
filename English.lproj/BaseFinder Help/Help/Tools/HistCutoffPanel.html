Cutoff Histogram
 

Sometimes the data will have some stray data points which will cause the automatic normalization of the data to malfunction. At such times the data will appear very shallow with just a few sharp peaks. While one can remove such aberrant regions through deletion, this is not always the most desirable method. This tool can also be useful for the removal of low amplitude noise in the same frequency range as dye-primer peaks that escapes removal through filtering (see Filtering and Matrixing ).

This tool operates by creating an ordered histogram of the data points. One can then remove the top x% and/or the bottom x% of data points.

This tool is best used after matrixing but before basecalling.

One can have the tool remove upper values only, lower values only, or both simultaneously. One can also apply this tool several times in succession by changing between upper and lower values and the cutoff threshold.

Threshold specifies what percent of the data to remove. Typical values are 1-2% for removal of upper values, and almost any value for removal of lower values (sometimes 80% is appropriate).

By default the tool is applied to all channels, but one can limit it to specific channels.

Possible side effects

Setting too high a threshold for removal of upper values can result in having peaks with flat tops. While this does not look good, it has little effect on the efficiency of the basecalling routines.

Setting too high a threshold of removal of lower values can result in the loss of low amplitude dye-primer peaks.
