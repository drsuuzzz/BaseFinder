Filter Convolution
 

The raw data coming off most automated sequencing hardware has both high and low frequency noise which makes basecalling of peaks difficult. This tool uses a published algorithm for using a gaussian filter to remove this extraneous noise.

Convolving is usually best done after the data has been matrixed and before the data is basecalled.

There are two different convolution algorithms implemented in this tool: Standard and Zero-Area. Standard convolution is functionally similar to a high frequency cutoff filter. Zero-area is functionally like a band pass filter where both high and low frequencies are filtered out.

Sigma should correspond to the Full Width at Half Maximum of a gaussian which represents a peak. This should be in data points. This corresponds to the high frequency cutoff point. The smaller the sigma, the higher the cutoff point, and the less that will be filtered out.

M is the window around the gaussian which is used. Since a guassian never reaches zero, a finite window of the gaussian must be specified. M should always be bigger than Sigma.

By default convolution is applied to all channels, but one can limit it only to specific channels.
