Mobility Shift
 

With fluorescent tagged DNA sequencing, the different dye molecules alter the speed at which the attached DNA fragments travel through the electrophoretic gel. This is due primarily to the fact that the various dyes have different sizes. This effects the spacing between peaks and, thus, basecalling accuracy.

This tool is best used after matrixing and smoothing (filtering, convolving, or deconvolving) but before basecalling. DO NOT MOBILITY SHIFT BEFORE MATRIXING!

Mobility Shift: Polynomial

Creating Mobility Shift Values:

To create new mobility shift values, first click on the New button. The mobility label will change to "Untitled-#", and a message will appear asking the user to select to enter equations by hand or to create by sliding traces. Press the OK button after making your selection.

Enter equations by hand. Enter the coefficient values for each channel by entering values into the a, b and c fields. By selecting the different channels, one can enter new coefficients corresponding to that channel. When finished select the Done button.

Create by sliding traces. First indicate the channel to slide. Then go to the data view window and click on the location you wish to slide. A segment of the trace will be highlighted. Drag this segment backwards or forwards the amount you wish to shift. In order for the tool to calculate the proper shift, do this at least three times with different data segments. The amount of shift and the origin point for the shift is collected. Generally one data channel is chosen as the reference by the user and all shifts are performed against this channel. However, this choice does not matter to the program. Once the user has finished entering shift values, they are tabulated, and the equation shown in the panel is fitted to the shift values for each channel. Channels for which no shifts were performed are assigned a shift polynomial with zero coefficients. When you have finished sliding traces for each of channel/dye types, click the Done button (alternate state of the new button).

Renaming Mobility Shift

After making a new mobility shift, it will be labeled as "Untitled-#". It is a good idea to rename the mobility shift to an appropriate label at this point. One does this by changing the name in the mobility label field and hitting return. One can also rename a mobility shift whenever they choose.

Each time the program starts up, new mobility shifts start with "Untitled-1" and progress from there until the program quits.

Applying mobility shift

Even if one has just created a new mobility shift, it still needs to be applied to the data. One does this by clicking on the Append button at the bottom of the Tool Inspector.

Removing mobility shifts

If mobility shifts are no longer useful, they can be removed from the list by first selecting the desired mobility shift from the popup list, and then hitting the Remove button.

Notes:

The mobility shifts are stored by default in the directory:

/Library/BaseFinder/MobilityFunctions

They are ascii formated files and can be easily copied. If you have difficulty saving or loading mobility shifts, it is probably a permissions problem in either creating this directory or writing to it.

If you want to put your MobilityFunctions directory somewhere other than /Library/BaseFinder, change the Resource Path field in the Preferences panel.
